🌙 DARK MODE IMPLEMENTATION
═════════════════════════════════════════════════════════════════════════════

How It Works:
1. User clicks Sun/Moon icon → toggleDarkMode() called
2. Zustand state updates
3. localStorage updated with preference
4. HTML root element gets "dark" class
5. Tailwind applies dark: prefixed styles

Global Styles (index.css):
├─ Dark mode conditional for all colors
├─ Smooth transitions (200ms)
├─ Custom dark variants for components
└─ Glass effect styling

Component Dark Support:
├─ bg-white dark:bg-slate-900
├─ text-slate-900 dark:text-white
├─ border-slate-200 dark:border-slate-800
├─ hover:bg-slate-100 dark:hover:bg-slate-800
└─ All interactive elements darkified

Persistence:
✓ Preference saved to localStorage
✓ Preference loaded on app startup
✓ Applied immediately without flashing


🔐 AUTHENTICATION & ROLE-BASED ACCESS
═════════════════════════════════════════════════════════════════════════════

Login/Register Flow:
User Input
  ↓
Form Validation
  ↓
API Call (POST /auth/login or /auth/register)
  ↓
Backend processes, returns {token, user}
  ↓
Zustand store updated (setToken, setUser)
  ↓
localStorage updated
  ↓
Redirect based on role


Protected Routes:
<ProtectedRoute requiredRole="auditor">
  <CompliancePage />
</ProtectedRoute>

Checks:
1. Is token present? → No → Redirect to /login
2. Is token valid? → Backend 401 response → Redirect to /login
3. Does role match? → No → Show "Access Denied" page
4. Otherwise → Render component


Role Permissions:
Administrator:     Full access to everything
Auditor:          View documents, create audits, generate reports
Document Manager:  Upload documents, manage workflows, set retention
Viewer:           Read-only access to documents and reports

Mobile First Approach:
1. Design for mobile (< 768px)
2. Add md: classes for tablets (768px+)
3. Add lg: classes for desktop (1024px+)

Common Patterns:
├─ flex-col md:flex-row           // Stack on mobile, row on desktop
├─ grid-cols-1 md:grid-cols-2 lg:grid-cols-4
├─ text-lg md:text-2xl lg:text-4xl

└─ hidden md:block                // Hide on mobile, show on desktop

Using Auth Store:
```javascript
import useAuthStore from './store/authStore';

function MyComponent() {
  const { user, token, logout, isDarkMode, toggleDarkMode } = useAuthStore();
  
  return (
    <div>
      <p>Welcome, {user?.fullName}</p>
      <p>Role: {user?.role}</p>

}
```

🎯 DEBUGGING TIPS
═════════════════════════════════════════════════════════════════════════════

Browser DevTools:
1. Application → localStorage: Check token, dark mode preference
2. Network tab: Watch API calls, check headers have Authorization
3. Console: Check for errors, CORS issues
4. React: React DevTools to inspect component state

Common Issues:

CORS Error:
→ Check backend .env has CORS_ORIGIN=http://localhost:3000
→ Verify backend is running

Login fails with network error:
→ Check .env.local has correct REACT_APP_API_URL
→ Verify backend running on port 4000

Blank page:
→ Hard refresh (Ctrl+Shift+R)
→ Check console for JavaScript errors

Token expired:
→ Redirects to /login automatically
→ User must login again
